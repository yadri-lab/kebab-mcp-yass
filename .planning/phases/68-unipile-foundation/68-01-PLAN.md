## Trust Boundaries

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-68-01-01 | Information Disclosure | manifest.ts testConnection logging | mitigate | Never log `dsn` or `token` value in `log.info(...)`. Log only the outcome string (e.g., `LinkedIn account count: 2`). Code review check on grep for `log.*UNIPILE_TOKEN` / `log.*token` in manifest.ts (must return zero). |
| T-68-01-02 | Tampering | registry.ts toolCount mismatch | mitigate | `toolCount: 0` in this plan, matches stub manifest's empty tools array. Verified by registry-metadata-consistency contract test before commit. Plan 06 bumps both atomically. |
| T-68-01-03 | Denial of Service | SDK install pulls 4.2 MB qrcode dep into lambda trace | accept | Connector is lazy-loaded (PERF-01); cost only paid when UNIPILE_* env vars present. size:check gate verifies /config eager bundle stays under 620 KB. |
| T-68-01-04 | Repudiation | testConnection silent failure (e.g., SDK throws on malformed DSN) | mitigate | testConnection wraps in try/catch and returns `{ ok: false, message: \`Unipile: \${toMsg(err)}\` }` — operator sees explicit failure in /config tile, not silent "disconnected" ambiguity. |
