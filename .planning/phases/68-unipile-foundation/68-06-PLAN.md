    - Handler flow (8 steps per RESEARCH §Code Examples):
      1. Generate `audit_id` (randomUUID via generateAuditId).
      2. Normalize profile_url. Compute `params_hash` from {tool: "linkedin_send_connection", profile_url_normalized, note: note ?? ""}.
      3. Call `checkDedup(params_hash)`. If hit: write a `dedup_hit: true` audit row, return envelope `{provider_ok: false, verified: false, dedup_hit: true, audit_id, crm_sync: 'pending'}` WITHOUT calling Unipile.
      4. Resolve `account_id` per D-20: if `args.account_id` provided, use it; else `client.account.getAll()` → filter `type === "LINKEDIN"`. Apply resolution rules below.
      5. Resolve `provider_id` via `resolveProviderId(profile_url, resolved_account_id)`.
      6. Write CRM outbox row via `crmBridge.writeOutbox(audit_id, {crm_log: args.crm_log ?? null})`.
      7. Call `withRetry(() => client.users.sendInvitation({account_id, provider_id, message?: note}))`. On error: classify, write audit row with `result`/`verified: false`, return envelope `{provider_ok: false, verified: false, dedup_hit: false, audit_id, error: <enum>, crm_sync: 'pending'}`.
      8. On send success: extract `invitation_id`. Call `pollForRelation(account_id, provider_id, [2000, 5000, 10000])`. Set `result: 'success'` if confirmed, `'unverified_timeout'` if not. Write audit row. Return envelope.
    - `pollForRelation` calls `client.users.getAllInvitationsSent({account_id})` between sleeps. Returns true if any `items[i].invited_user_id === provider_id`. Catches transient errors in each poll iteration (do NOT abort polling on a single failure).
    - `resolveAccountId(args, client)`: if `args.account_id` provided → return it. Else fetch `account.getAll()` → filter LinkedIn type. 0 → throw `{error: "error_no_linkedin_account"}`; 1 → return the one id; ≥2 → throw `{error: "error_account_id_required", available_accounts: [...ids]}`.

    ```typescript
    import { z } from "zod";
    import type { ToolResult } from "@/core/types";
    import { getUnipileClient } from "../client";
    import { withRetry } from "../lib/retry";
    import { resolveProviderId, normalizeProfileUrl } from "../lib/identifiers";
    import {
      computeParamsHash,
      checkDedup,
      writeAuditRow,
      generateAuditId,
      type AuditResult,
    } from "../lib/audit";
    import { crmBridge } from "../lib/crm-bridge";
    import { classifyUnipileError } from "../lib/errors";

    export const linkedinSendConnectionSchema = {
      profile_url: z
        .string()
        .url()
        .describe("Public LinkedIn profile URL (https://linkedin.com/in/<slug>; locale prefixes accepted)."),
      note: z
        .string()
        .max(300)
        .optional()
        .describe("Optional connection request note (≤300 chars LinkedIn cap). Changing 1 char in the note bypasses dedup — this is intentional (D-05)."),
      account_id: z
        .string()
        .optional()
        .describe("Unipile LinkedIn account_id. Optional — if exactly one LinkedIn account is connected, it is used silently. If multiple, account_id is required."),
      actor_user_id: z
        .string()
        .describe("User id of the operator triggering the call. Recorded in the audit log."),
      crm_log: z
        .record(z.string(), z.unknown())
        .optional()
        .describe("Free-form CRM payload passed to the outbox row. Phase 70 will POST this to UNIPILE_CRM_WEBHOOK_URL."),
    };

    type SendArgs = {
      profile_url: string;
      note?: string;
      account_id?: string;
      actor_user_id: string;
      crm_log?: Record<string, unknown>;
    };

    interface SendEnvelope {
      provider_ok: boolean;
      verified: boolean;
      crm_sync: "pending";              // D-01: hardcoded in phase 68
      dedup_hit: boolean;
      audit_id: string;
      invitation_id?: string;
      error?: string;
      available_accounts?: string[];    // populated on error_account_id_required (D-20)
    }

    function envelope(e: SendEnvelope): ToolResult {
      return {
        content: [{ type: "text" as const, text: JSON.stringify(e, null, 2) }],
      };
    }

    /**
     * D-20 account_id resolution.
     * Returns the account_id to use, or throws an error envelope describing the resolution failure.
     */
    async function resolveAccountId(args: SendArgs): Promise<{ accountId: string } | { error: "error_no_linkedin_account" } | { error: "error_account_id_required"; available_accounts: string[] }> {
      if (args.account_id) return { accountId: args.account_id };
      const resp = await getUnipileClient().account.getAll();
      const items = (resp as { items?: Array<{ id: string; type: string }> }).items ?? [];
      const linkedinAccounts = items.filter((i) => i.type === "LINKEDIN").map((i) => i.id);
      if (linkedinAccounts.length === 0) return { error: "error_no_linkedin_account" };
      if (linkedinAccounts.length > 1) return { error: "error_account_id_required", available_accounts: linkedinAccounts };
      return { accountId: linkedinAccounts[0]! };
    }

    /**
     * D-13 verify-after-write: 3 polls at 2s/5s/10s (~17s total budget).
     * Confirms send by listing sent invitations and checking invited_user_id === provider_id.
     * Per Assumption A3: if Unipile's propagation latency exceeds 17s consistently,
     * this returns false even on real success — re-evaluate then.
     */
    async function pollForRelation(
      accountId: string,
      providerId: string,
      delaysMs: number[]
    ): Promise<boolean> {
      const client = getUnipileClient();
      for (const delay of delaysMs) {
        await new Promise((r) => setTimeout(r, delay));
        try {
          const invitations = await client.users.getAllInvitationsSent({ account_id: accountId });
          const items = (invitations as { items?: Array<{ invited_user_id: string | null }> }).items ?? [];
          if (items.some((i) => i.invited_user_id === providerId)) return true;
        } catch {
          // D-16: transient poll error is not fatal — continue to next delay.
        }
      }
      return false;
    }

    export async function handleLinkedinSendConnection(args: SendArgs): Promise<ToolResult> {
      const auditId = generateAuditId();
      const profileUrlNormalized = (() => {
        try { return normalizeProfileUrl(args.profile_url); }
        catch { return args.profile_url; } // fall through to SDK to produce a meaningful error
      })();
      const paramsHash = computeParamsHash({
        tool: "linkedin_send_connection",
        profile_url_normalized: profileUrlNormalized,
        note: args.note ?? "",
      });

      // 1. Dedup check (D-05/D-06)
      const dup = await checkDedup(paramsHash);
      if (dup) {
        await writeAuditRow({
          audit_id: auditId,
          actor_user_id: args.actor_user_id,
          tool: "linkedin_send_connection",
          account_id: args.account_id ?? dup.account_id,
          params_hash: paramsHash,
          result: dup.result,        // mirror prior result for trace continuity
          verified: dup.verified,
          dedup_hit: true,
          timestamp: new Date().toISOString(),
        });
        return envelope({
          provider_ok: false,
          verified: false,
          crm_sync: "pending",
          dedup_hit: true,
          audit_id: auditId,
        });
      }

      // 2. Resolve account_id (D-20)
      const acct = await resolveAccountId(args);
      if ("error" in acct) {
        const result: AuditResult = "error_account_restricted"; // D-20 errors classify as 'restricted' in the audit enum
        await writeAuditRow({
          audit_id: auditId,
          actor_user_id: args.actor_user_id,
          tool: "linkedin_send_connection",
          account_id: args.account_id ?? "",
          params_hash: paramsHash,
          result,
          verified: false,
          dedup_hit: false,
          timestamp: new Date().toISOString(),
        });
        const env: SendEnvelope = {
          provider_ok: false,
          verified: false,
          crm_sync: "pending",
          dedup_hit: false,
          audit_id: auditId,
          error: acct.error,
        };
        if ("available_accounts" in acct) env.available_accounts = acct.available_accounts;
        return envelope(env);
      }
      const accountId = acct.accountId;

      // 3. Resolve provider_id (KV cache + SDK fallback). On 429: surface error.
      let providerId: string;
      try {
        const r = await resolveProviderId(args.profile_url, accountId);
        providerId = r.provider_id;
      } catch (err) {
        const result = classifyUnipileError(err);
        await writeAuditRow({
          audit_id: auditId,
          actor_user_id: args.actor_user_id,
          tool: "linkedin_send_connection",
          account_id: accountId,
          params_hash: paramsHash,
          result,
          verified: false,
          dedup_hit: false,
          timestamp: new Date().toISOString(),
        });
        return envelope({ provider_ok: false, verified: false, crm_sync: "pending", dedup_hit: false, audit_id: auditId, error: result });
      }

      // 4. Outbox row (D-01: skeleton only)
      await crmBridge.writeOutbox(auditId, { crm_log: args.crm_log ?? null });

      // 5. Send invitation (with retry on 429/5xx)
      let invitationId: string | undefined;
      try {
        const resp = await withRetry(() =>
          getUnipileClient().users.sendInvitation({
            account_id: accountId,
            provider_id: providerId,
            ...(args.note ? { message: args.note } : {}),
          })
        );
        invitationId = (resp as { invitation_id?: string }).invitation_id;
      } catch (err) {
        const result = classifyUnipileError(err);
        await writeAuditRow({
          audit_id: auditId,
          actor_user_id: args.actor_user_id,
          tool: "linkedin_send_connection",
          account_id: accountId,
          params_hash: paramsHash,
          result,
          verified: false,
          dedup_hit: false,
          timestamp: new Date().toISOString(),
        });
        return envelope({ provider_ok: false, verified: false, crm_sync: "pending", dedup_hit: false, audit_id: auditId, error: result });
      }

      // 6. Verify-after-write (D-13: 3 polls @ 2s/5s/10s)
      const verified = await pollForRelation(accountId, providerId, [2000, 5000, 10000]);
      const result: AuditResult = verified ? "success" : "unverified_timeout";

      // 7. Write final audit row
      await writeAuditRow({
        audit_id: auditId,
        actor_user_id: args.actor_user_id,
        tool: "linkedin_send_connection",
        account_id: accountId,
        params_hash: paramsHash,
        result,
        verified,
        dedup_hit: false,
        timestamp: new Date().toISOString(),
      });

      // 8. Return envelope
      const out: SendEnvelope = {
        provider_ok: true,
        verified,
        crm_sync: "pending",
        dedup_hit: false,
        audit_id: auditId,
      };
      if (invitationId) out.invitation_id = invitationId;
      if (!verified) out.error = "unverified_timeout";
      return envelope(out);
    }
    ```

    import { z } from "zod";
    import type { ToolResult } from "@/core/types";
    import { getUnipileClient } from "../client";
    import { withRetry } from "../lib/retry";
    import { resolveProviderId, normalizeProfileUrl } from "../lib/identifiers";
    import { classifyUnipileError } from "../lib/errors";

    export const linkedinGetRelationshipStatusSchema = {
      profile_url: z
        .string()
        .url()
        .describe("Public LinkedIn profile URL. Returns the network distance (1/2/3/null) of this profile relative to the account."),
      account_id: z
        .string()
        .optional()
        .describe("Unipile LinkedIn account_id. Optional — if exactly one LinkedIn account is connected, it is used silently."),
    };

    type GetRelArgs = {
      profile_url: string;
      account_id?: string;
    };

    interface RelStatusEnvelope {
      degree: 1 | 2 | 3 | null;       // D-21: 1|2|3|null only
      connection_status: string;      // raw network_distance value, or "unknown"
      error?: string;
      available_accounts?: string[];
    }

    function envelope(e: RelStatusEnvelope): ToolResult {
      return { content: [{ type: "text" as const, text: JSON.stringify(e, null, 2) }] };
    }

    /** D-20 account_id resolution — same rules as linkedin_send_connection. */
    async function resolveAccountId(args: GetRelArgs):
      Promise<{ accountId: string } | { error: "error_no_linkedin_account" } | { error: "error_account_id_required"; available_accounts: string[] }> {
      if (args.account_id) return { accountId: args.account_id };
      const resp = await getUnipileClient().account.getAll();
      const items = (resp as { items?: Array<{ id: string; type: string }> }).items ?? [];
      const linkedinAccounts = items.filter((i) => i.type === "LINKEDIN").map((i) => i.id);
      if (linkedinAccounts.length === 0) return { error: "error_no_linkedin_account" };
      if (linkedinAccounts.length > 1) return { error: "error_account_id_required", available_accounts: linkedinAccounts };
      return { accountId: linkedinAccounts[0]! };
    }

    /**
     * Map Unipile's network_distance string to the public 1|2|3|null degree.
     * Per Pitfall 3: missing field is NOT "third degree" — it is null (unknown / private profile / sparse data).
     */
    function mapDegree(networkDistance: string | undefined): 1 | 2 | 3 | null {
      switch (networkDistance) {
        case "FIRST_DEGREE": return 1;
        case "SECOND_DEGREE": return 2;
        case "THIRD_DEGREE": return 3;
        case "OUT_OF_NETWORK": return null;
        default: return null;
      }
    }

    export async function handleLinkedinGetRelationshipStatus(args: GetRelArgs): Promise<ToolResult> {
      const acct = await resolveAccountId(args);
      if ("error" in acct) {
        const env: RelStatusEnvelope = { degree: null, connection_status: "unknown", error: acct.error };
        if ("available_accounts" in acct) env.available_accounts = acct.available_accounts;
        return envelope(env);
      }
      const accountId = acct.accountId;

      // Resolve provider_id (and incidentally warm the URN cache). On 429 / error: surface.
      let providerId: string;
      try {
        const r = await resolveProviderId(args.profile_url, accountId);
        providerId = r.provider_id;
      } catch (err) {
        return envelope({ degree: null, connection_status: "unknown", error: classifyUnipileError(err) });
      }

      // Call getProfile to read network_distance. The slug is the identifier.
      const slug = normalizeProfileUrl(args.profile_url).slice("https://linkedin.com/in/".length);
      try {
        const profile = await withRetry(() =>
          getUnipileClient().users.getProfile({ account_id: accountId, identifier: slug })
        );
        const nd = (profile as { network_distance?: string }).network_distance;
        return envelope({
          degree: mapDegree(nd),
          connection_status: nd ?? "unknown",
        });
      } catch (err) {
        return envelope({ degree: null, connection_status: "unknown", error: classifyUnipileError(err) });
      }
      // providerId is intentionally unused after resolveProviderId — its side effect of cache warming is the benefit.
    }

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP client (LLM) → tool handler | LLM-provided args validated by zod schema at MCP transport layer. |
| Tool handler → Unipile API (via SDK) | All write calls flow through getUnipileClient + withRetry; auth via X-API-KEY set by SDK. |
| Tool handler → KV (audit + outbox + URN cache) | Tenant-scoped via getContextKVStore. |
| Verify-after-write polling | Bounded to 3 polls / ~17s total — well under Vercel 60s lambda budget. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-68-06-01 | Tampering | LLM omits `actor_user_id` to avoid audit trail | mitigate | zod schema marks `actor_user_id` as REQUIRED — request fails validation at MCP transport before handler runs. |
| T-68-06-02 | Tampering | LLM forges a `verified: true` envelope by manipulating tool arguments | mitigate | envelope is constructed server-side based on actual poll results — no caller-supplied field bypasses this. |
| T-68-06-03 | Information Disclosure | Audit log written even on dedup hit → reveals prior attempt to LLM | accept | The dedup envelope returns `dedup_hit: true` deliberately — the LLM should know NOT to retry with the same note. The prior `result` is mirrored from the cached row for trace continuity. Risk: an attacker tenant could infer prior activity, but tenant scoping prevents cross-tenant inference. |
| T-68-06-04 | Repudiation | Dedup-hit path doesn't write a new audit row → operator sees only the original | mitigate | We DO write a new audit row on dedup hit (with `dedup_hit: true`, new `audit_id`, same `params_hash`, mirrored `result`/`verified`) — operator sees BOTH the original send and each repeat attempt. Verified by code path in handler. |
| T-68-06-05 | Denial of Service | Verify-after-write polling consumes a Unipile read quota call per attempt | accept | At most 3 read calls per send. At Cadens scale (5-30 sends/day), this is 15-90 extra read calls — negligible vs Unipile's typical quotas. |
| T-68-06-06 | Elevation of Privilege | `account_id` from one tenant used to send via another tenant's LinkedIn account | accept | account_id is operator-scoped (one UNIPILE_TOKEN per Kebab deploy). Multi-tenant Kebab deploys with shared Unipile tokens are out of scope — each tenant should have its own deploy. Phase 71 may add per-tenant Unipile token scoping if needed. |
| T-68-06-07 | Tampering | Note text manipulated to bypass dedup but contains malicious payload (e.g., zero-width chars) | mitigate | Hash is over the raw note string — any byte-level change = new hash = new call. This is by design (D-05). LinkedIn-side anti-spam handles the malicious content angle, not Kebab. |
